1. IT 프로젝트 전과정
- IT 프로젝트 전과정 및 각 단계별 기본 업무를 하나의 장표에 도식화 했습니다. 

- IT 프로젝트 진행순서는 기본적으로 

프로젝트 제안 및 수주 => 프로젝트 계획 및 시작 => 기획 => 디자인 => 퍼블리싱 => 개발 => 테스트 => 오픈 및 안정화 (검수 포함)의 단계를 가지고 있습니다. 

2. 각 과정별 상세 내용

1) 프로젝트 제안 및 수주 단계

  a. 사전분석 : 프로젝트 발주를 위한 고객사의 사전 분석 단계 (RFI 포함)
  b. 사업 공고 및 제안 요청 : 사전 분석이 끝난 고객사가 제안 및 발주를 하는 단계 (RFP 포함)
  c. 제안 및 견적 : 사업공고를 보고, 프로젝트 진행에 대한 견적 및 제안 단계 (RFQ 포함)
  d. 낙찰 : 고객이 최종으로 업체를 선정 및 계약

2) 프로젝트 계획 및 시작

  a. 팀구성 : 프로젝트 수주후 급하게 개발을 진행할 팀을 구성
  b. 프로젝트 진행 전반 계획 : 계약 이후 고객사에 들어가서 현업 TFT와 진행에 대한 기본 계획 수립
  c. 진행일정 구성 : 상세한 요구사항 분석 전에, 고객사의 RFP 정보를 기준으로 간략한 개발 일정 수립
  d. 프로젝트 본격 시작 : 착수 보고 진행후 프로젝트를 본격적으로 진행

3) 기획

  a. 요구사항 분석 : 현업 담당자와 함께 개발할 시스템의 요구사항을 분석
  b. 기능개발 정의 : 개발할 기능에 대한 정의 및 개발 일정 수립 (WBS 작성)
  c. 프로세스 정의 : 분석된 요구사항에 대한 시스템 프로세스 정리
  d. 상세화면 기획 : 분석된 요구사항 기반 실제 프로그램을 개발하기 위한 기획 작업(화면을 그리는 작업)

4) 디자인 

  a. 디자인 컨셉 확인 : 고객이 원하는 디자인의 컨셉을 확인
  b. 디자인 시안 작업 : 고객의 컨셉에 따라서 3가지 정도의 디자인 예시를 만드는 과정, 보통 5개 정도 화면 대상으로 시안을 만듬(로그인, 메인화면, 리스트 화면, 상세화면, 특별한 샘플화면 정도)
  c. 시안 확정 및 Develop : 시안중에서 고객이 원하는 안을 확정, 그리고 그 시안을 좀 더 디테일하게 발전
  d. 전체 화면 디자인 작업 : 디자인 시안을 기준으로 기획문서의 전체 화면을 디자인 하는 작업


5) 퍼블리싱 

  a. 일정에 따른 퍼블리싱 : 디자이너가 작업해준 디자인 화면을 개발자가 개발 할 수 있도록 전처리 작업
  b. 유저 SIDE 우선 작업 : 작업 일정이 바쁠 경우 유저가 보는 쪽의 화면 작업을 우선
  c. 관리 SIDE 다음 작업 : 유저 SIDE가 끝나면 관리쪽 작업
  d. 퍼블리싱 보안 지원 : 완료된 퍼블리싱 페이지로 개발자가 개발시, 수정 보완 요청에 대한 대응

6) 개발 

  a. (사전) 서버-네트워크 환경 구성 : 프로젝트 초반 단계부터 현업IT 팀과 상의해서 서버 및 네트워크 특히 하드웨어 관련 환경 사전 구축
  b. (사전) 개발 환경 구성 : 개발을 위한 프레임워크 부터 전체 시스템에 대한 기본 설계 및 개발을 할 수 있는 환경 구성
  c. 개발 : 실제 기능을 개발 
    => APP 및 서버 개발로 구분
    => APP은 안드로이드와 iOS로 구분
    => 서버 개발은 Front End, Back End로 구분

7)  테스트 

  a. 테스트 케이스 작성 : 테스트를 위한 기본 케이스 작성
  b. 테스트 시나리오 작성 : 테스트 케이스를 기초로 테스트를 위한 시나리오 작성
  c. 단위 테스트 및 보완 : 각 단위 기능에 대한 테스트 및 보완
  d. 통합 테스트 및 보완 : 단위 테스트 및 보완이 끝나고, 전체 프로세스의 흐름을 통합하여 테스트 및 보완

8) 오픈 및 안정화

  a. 사전 교육 : 개발 시스템에 대한 메뉴얼 작성 및 사용자 교육
  b. 오픈 준비 : 오픈을 위한 시스템 준비부터 업무준비까지
  c. 오픈 : 서비스에 대한 오픈
  d. 안정화 : 오픈 이후 일정 기간동안 시스템이 정상적으로 잘 돌아가는지 확인 및 지원하는 단계, 이때 이상이 없으면 검수 진행
